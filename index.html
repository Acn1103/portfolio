⚡ How live updates work

Credentials must be baked into the deployed file so every visitor’s browser can read your Supabase data — not just yours.

  1. Enter your URL & key below → Test Connection until you see ✅
  2. Click “Download Configured File” — credentials are baked into index.html
  3. Deploy that file to Netlify — one time only
  4. After that, every change you make via this admin appears instantly for all visitors — no further deploys ever

🔗 Supabase Connection

After connection shows ✅: download the configured file and deploy it to Netlify once.

📋 One-Time Setup — follow in order

1

Get your Supabase URL & API key

Supabase Dashboard → your project → Settings → API. Copy the Project URL (top of page) and try the Publishable key first. If that fails, use the Legacy anon key (second tab).

2

Run this SQL once in Supabase SQL Editor

-- Supabase Dashboard → SQL Editor → New Query → paste → Run create table if not exists projects ( id bigint primary key generated always as identity, title text not null, description text default '', year integer default 2025, type text default 'website', github_url text default '', live_url text default '', color text default '#3b82f6', image_url text, created_at timestamptz default now() ); create table if not exists about_me ( id integer primary key default 1, bio1 text default '', bio2 text default '' ); create table if not exists skills ( id bigint primary key generated always as identity, name text not null, pct integer default 50, sort_order integer default 0 ); create table if not exists settings (key text primary key, value text); insert into about_me (id,bio1,bio2) values (1, 'Hello! I''m Arnold, an IT student with 3+ years of experience in digital design and development.', 'Exploring where I fit best in tech — ML, full-stack, or something I haven''t tried yet.' ) on conflict (id) do nothing; alter table projects enable row level security; alter table about_me enable row level security; alter table skills enable row level security; alter table settings enable row level security; create policy "public read" on projects for select using (true); create policy "anon write" on projects for all using (true) with check (true); create policy "public read" on about_me for select using (true); create policy "anon write" on about_me for all using (true) with check (true); create policy "public read" on skills for select using (true); create policy "anon write" on skills for all using (true) with check (true); create policy "public read" on settings for select using (true); create policy "anon write" on settings for all using (true) with check (true); insert into storage.buckets (id,name,public) values ('portfolio-media','portfolio-media',true) on conflict do nothing; create policy "public read" on storage.objects for select using (bucket_id='portfolio-media'); create policy "anon upload" on storage.objects for insert with check (bucket_id='portfolio-media'); create policy "anon update" on storage.objects for update using (bucket_id='portfolio-media'); create policy "anon delete" on storage.objects for delete using (bucket_id='portfolio-media');
3

Test Connection → see ✅ "Connected! Tables found"

Add your content (projects, photos, bio, skills, CV). Then click "Download Configured File" above to get index.html with your credentials baked in.

4

Deploy index.html to Netlify — once, then never again

netlify.com → drag index.html onto the deploy area → done. From this point, every admin change you make is instantly visible to all visitors worldwide — no further deploys needed, ever.
